Data Flow — Full Apply Pipeline

From search page to submitted application

sequenceDiagram
    actor User
    participant PW as page-world.js
    participant MAIN as main.js
    participant MPH as main-page-handlers
    participant VL as vacancy-list.js
    participant MS as match-scorer.js
    participant Panel as UI Panel
    participant EO as apply-orchestrator
    participant RL as rate-limiter
    participant EA as apply-actions
    participant STG as storage
    participant BG as background.js

    User->>PW: Opens hh.ru search
    PW->>MAIN: hh-ar-spa-navigate
    MAIN->>MPH: initPageLogic()
    MPH->>VL: parseVacanciesFromPage()
    VL->>MS: computeMatchScore(resume, vacancy)
    MS-->>VL: score + breakdown
    VL-->>MPH: scored vacancy list
    MPH->>Panel: updateVacancies()
    Panel-->>User: Score cards in sidebar

    User->>Panel: Click "Apply" or auto-apply
    Panel->>EO: applyToVacancy(id) / applyToAll()
    EO->>RL: check() — rate limit
    RL-->>EO: allowed / denied
    EO->>STG: isAlreadyApplied(id)?
    STG-->>EO: false
    EO->>EA: waitForPageReady()
    EA->>EA: clickApplyButton()
    EA->>EA: waitForPopupAndSubmit()
    EA-->>EO: success
    EO->>STG: markAsApplied(id)
    EO->>BG: incrementApplied()

    Note over EO,RL: Adaptive: burst detection,
typing simulation, random delays
