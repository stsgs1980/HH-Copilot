- **Date:** 2026-06-23
- **Task:** F4.1 (cascade/state.json, P4 phase)
- **Version bump:** 1.9.48.0 → 1.9.49.0 (Rule 9.2, before feat commit)
- **Implements (function inventory):** F-NG-01 (chat list), F-NG-02 (unread badge), F-NG-03 (status categorization), F-NG-05 (refresh button)
- **Approach chosen:** Variant 3 — hybrid from existing parser data

## 1. Problem & Constraint

The hh.ru negotiations list page (`/applicant/negotiations`) does **not** expose
the last-message preview or a per-conversation unread count. This is documented
in `docs/research/04-negotiations-dom-analysis.md` (Key Decision #1): the list
page yields only vacancy title, company, application date, and status
(`not-viewed`/`viewed`/`discard`/`invite`). Real chat previews require the
`chatik` interface and live DOM research — deferred to the new backlog task
**F4.5**.

F4.1 therefore closes its acceptance criteria with a **hybrid** built purely
from fields the existing `parseSingleItem()` already returns (`status`,
`statusText`, `date`, `unread`). No parser changes, no new selectors, no
hallucinated data sources.

## 2. What Already Works (do not touch)

- F1.9 renders the list with status/tab filters, refresh button, empty/error/
  loading states (`src/ui/tabs/negotiations.js`).
- Click on an item already opens `#neg-chat-area` via `selectConversation`
  → `renderChatMessages` (`src/ui/panel/helpers.js:37`,
  `src/ui/panel/sidebar-events.js:155`). **Requirement 2 is satisfied as-is.**
- Status badge (color-coded, from `STATUS_CONFIG`) already present.

## 3. Item Layout (target)

```
+---------------------------------------------------+
| o  AI Engineer            [Приглашение] [all]      |  title row (o = unread dot)
|    Matuzova Olesya V.                              |  company (existing)
|    Приглашение                            вчера    |  preview + relative ts (NEW)
+---------------------------------------------------+
```

### 3.1 Field rendering rules

| Field | Source | Rule |
|-------|--------|------|
| Unread dot | `status === 'not-viewed' \|\| status === 'invite'` | Red dot left of title; **hidden otherwise**. `aria-label="Есть непрочитанные"`, `title="Непросмотрено"`. Dot, not a number — we have no real count. |
| Preview line | `statusText` | If `statusText` is `undefined`/`null`/`''` (parse failure) → render **"(нет сообщений)"** in grey. Otherwise render `statusText` in normal text color. |
| Timestamp | `c.date` → `formatRelativeTime()` | If result is `''` (unrecognized date), show **no** separator and **no** timestamp. Otherwise show `· {relativeTime}`. |

### 3.2 Long-text handling (anti-hallucination F4.1)

Preview line uses `white-space:nowrap; overflow:hidden; text-overflow:ellipsis`
so long status text truncates with an ellipsis rather than wrapping/breaking layout.

## 4. New utility: `formatRelativeTime`

Pure function, no side effects. Lives in a dedicated file because both target
files (`negotiations.js` 237 lines, `negotiations.js` parser 239 lines) are near
the AHG HARD limit of 250.

**File:** `src/ui/tabs/negotiations-format.js`

**Signature:**
```js
/**
 * Convert an hh.ru negotiations date string to a short relative label.
 *
 * Recognized inputs:
 *   - "вчера"                       -> "вчера"
 *   - "сегодня"                     -> "сегодня"
 *   - bare time "14:30"             -> "" (no calendar date; we have no "today"
 *                                       certainty without a date — return empty
 *                                       rather than guess)
 *   - "9 июня", "5 июня 2025"       -> as-is (month names vary; parsing year
 *                                       for "2ч назад" requires a full timestamp
 *                                       we do not have)
 *   - relative already ("2ч назад") -> returned unchanged
 *   - "", undefined, null           -> ""
 *   - unrecognized string           -> ""
 *
 * NEVER returns undefined. NEVER throws.
 * @param {string} dateStr
 * @param {Date} [now=new Date()]
 * @returns {string}
 */
export function formatRelativeTime(dateStr, now = new Date()) { ... }
```

**Important:** hh.ru list-page dates are calendar strings like "9 июня" without
a time component, so we cannot reliably compute "2ч назад". The function
therefore passes recognized human strings through ("вчера", "сегодня",
already-relative), and returns `''` for anything it cannot safely interpret.
This avoids hallucinating a precise time. (Real relative timestamps are part
of F4.5 where the chatik interface provides full timestamps.)

This keeps the function simple, deterministic, and unit-testable.

## 6. `renderNegotiationItem(c)` contract

**Input:** a single parsed negotiation object from `panelState.negotiations`
(produced by the aggregator). May be missing optional fields.

**Output:** an HTML string for one `.conv-item` row. Structure:
- Outer `<div class="conv-item" data-conv-id tabindex role>`
  - Left column `<div style="flex:1;min-width:0;">`
    - Title row: `[unread dot?] vacancyTitle [statusBadge] [tabBadge] [alsoIn]`
    - Company row: `company [· date?]`  (date here is raw `c.date`, kept as before)
    - **Preview row (NEW):** `previewContent [· relativeTime?]`

**Preview content resolution (edit #3):**
```js
const hasStatusText = c.statusText !== undefined && c.statusText !== null && c.statusText !== '';
const previewText = hasStatusText ? c.statusText : '(нет сообщений)';
const previewStyle = hasStatusText
  ? 'font-size:11px;color:#71717A;white-space:nowrap;overflow:hidden;text-overflow:ellipsis;'
  : 'font-size:11px;color:#a1a1aa;font-style:italic;white-space:nowrap;overflow:hidden;text-overflow:ellipsis;'; // grey + italic for "(нет сообщений)"
```
Grey + italic applied **only** to the "(нет сообщений)" fallback. Normal
`statusText` uses the standard muted color. (Future F4.5: when `c.lastMessage`
is present, it takes precedence over `statusText`.)

**Escape discipline:** all interpolated values pass through `esc()` (existing
`src/ui/html.js`), matching the current template.

## 8. Anti-hallucination checklist (F4.1 acceptance)

- [x] Empty list → placeholder ("Откликов пока нет" / error variant) — already handled, unchanged.
- [x] Long text → ellipsis (CSS `text-overflow:ellipsis` on preview line).
- [x] Status from predefined list (`STATUS_CONFIG`; unknown → grey "Неизвестно").
- [x] No fabricated selectors — parser untouched.
- [x] No fabricated fields — uses only existing `status`, `statusText`, `date`.
- [x] Unread = dot, not a fake number.
- [x] `formatRelativeTime` returns `''` (never `undefined`) on unrecognized input.
- [x] Empty timestamp → no separator, no timestamp (no stray "·").
- [x] "(нет сообщений)" only on parse failure (`statusText` falsy); normal color otherwise.
