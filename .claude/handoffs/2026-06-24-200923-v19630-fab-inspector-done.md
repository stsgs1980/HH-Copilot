## Codebase Understanding

### Architecture Overview

HH Copilot is a Chrome extension that automates job applications on hh.ru.
The codebase is split into:

- **`extension/src/`** — TypeScript-free ES modules. Strict anti-monolith
  rule (AHG Rule 12): every file ≤200 lines, hard limit.
- **`extension/src/ui/`** — UI layer. `fab.js` (main FAB), `fab-inspector-button.js`
  (NEW in v1.9.63.0, the purple mini-button), `dom-inspector.js` + panel/report
  (visual element inspector for hh.ru), `panel/` (sidebar orchestration),
  `state.js` (shared mutable state + accessor functions), `tabs/` (sidebar tabs).
- **`extension/src/lib/`** — Pure libs: selectors, anti-hallucination helpers,
  storage, timing, rate-limiter, tour-engine.
- **`extension/background/`** — Service worker.
- **`cascade-state.json`** (repo root) — Single source of truth for cascade
  tasks. Schema: `{ _meta, functionInventory, phases[] }`. Tasks live at
  `.phases[].tasks[]`. **DO NOT overwrite this file** — it was corrupted
  once before (see v1.9.62.0 CHANGELOG) and the restoration cost a full
  session.
- **`anti-hallucination-guard/`** — Git submodule. **IMMUTABLE** (Rule 16).
  Never commit pointer changes to this. Its scripts (validate.sh, discover,
  verify) are for module-internal use; the HH-Copilot repo uses
  `scripts/ahg.sh` as the wrapper.
- **`scripts/ahg-bump-safe.sh`** — Safe version-bump wrapper. Use this
  INSTEAD of `ahg bump` directly. The raw `ahg bump` would clobber version
  strings in 25+ foreign skill files because AHG's `discover-versions.ts`
  SKIP_DIRS omits `skills/`, `FabInspector/`, `hh-extension/`.
- **`worklog.md`** (repo root, 4346 lines) — Append-only shared log. Every
  task gets a new section starting with `---`. NEVER rewrite history.
  Pre-commit hook rejects commits if worklog.md hasn't been touched in
  the last 10 minutes.

### Key Patterns Discovered

- **All FAB/overlay elements live in MAIN document** (not Shadow DOM) so they
  can cover the whole page. CSS uses `style.setProperty(prop, value, 'important')`
  to resist hh.ru style overrides.
- **`shouldIgnore(el)` in dom-inspector.js** lists element IDs the inspector
  must NOT highlight (its own UI). The mini-button ID `hh-ar-fab-inspector`
  is already in that list — do not remove it.
- **Inspector `toggleInspector(btn)` visual update** assumes a transparent
  header button. When called from the FAB mini-button, do NOT pass `btn` —
  pass nothing and manage the pressed-state externally via
  `setFabInspectorActive(isInspectorActive())`.
- **Pre-commit hook chain**: anti-monolith check → co-change check →
  worklog freshness check (10 min). The worklog check is the one that bites
  most often — append a `---` section right before committing.
- **Version bump workflow**:
  1. `bash scripts/ahg-bump-safe.sh X.Y.Z` (updates ~15 files + CHANGELOG placeholder)
  2. Manually expand the CHANGELOG placeholder with real Added/Changed/Fixed sections
  3. `cd extension && npm run lint:ci && npm test && npm run build`
  4. `bash scripts/ahg.sh verify --ci`
  5. Append fresh worklog entry (≤10 min before commit)
  6. `git add -A && git commit`

## Git credential setup (current)

**State**: HTTPS remote + `git credential.helper cache` (24h in-memory TTL).
No plaintext PAT in `.git/config` or `.gitmodules`.

**What's in `.git/config`:**
```
[remote "origin"]
    url = https://github.com/stsgs1980/HH-Copilot.git
[submodule "anti-hallucination-guard"]
    url = https://github.com/stsgs1980/Anti-hallucination-guard.git
[credential]
    helper = cache --timeout=86400
```

**How to re-prime the cache when PAT expires (HTTP 401 on push):**
```bash
# User supplies fresh PAT via upload file or chat
PAT="$(tr -d '\n\r' < /home/z/my-project/upload/HH-Copilot\ git.txt | sed 's|^https://||;s|@.*||')"
printf 'protocol=https\nhost=github.com\nusername=stsgs1980\npassword=%s\n\n' "$PAT" \
  | git credential approve
# Verify:
printf 'protocol=https\nhost=github.com\n\n' | git credential fill
# Should print: protocol=https / host=github.com / username=stsgs1980 / password=<PAT>
```

**Why not SSH:** `openssh-client` is not installed in the sandbox and
there is no root to install it. `credential.helper cache` is the next-best
option -- PAT lives only in memory, dies when the cache daemon exits or
after 24h TTL.

**Why not `credential.helper store` (plaintext on disk):** Would re-create
the original problem -- PAT in a file. Cache is in-memory only.

**Push workflow from sandbox:**
```bash
cd /home/z/my-project/HH-Copilot-repo
# (worklog.md must be updated within last 10 min -- pre-commit hook checks)
git add -A
git restore --staged anti-hallucination-guard  # NEVER commit submodule pointer by accident
git commit -m "<conventional commit message>"
git push origin main
```

### Potential Gotchas

- **`npm` runs in `extension/`, not repo root.** Repo root has no package.json.
  Always `cd /home/z/my-project/HH-Copilot-repo/extension && npm ...`.
- **`scripts/ahg.sh` runs from repo root**, NOT extension/.
- **Anti-monolith Rule 12 limit is 200 lines** (warning) / 250 lines (error).
  If a file crosses 200, split it. The pre-commit hook will warn but not block.
- **Co-change check** in pre-commit hook may complain if you edit a file
  without also editing its "co-change group" siblings. Usually harmless,
  but if it blocks, check the message for which sibling it expects.
- **`toggleInspector(btn)` clobbers purple background** if you pass the FAB
  mini-button as `btn`. Its built-in visual update sets `btn.style.background = 'transparent'`
  and `btn.style.color = '#52525b'` — fine for header button, wrong for
  FAB mini-button. Call `toggleInspector()` with NO arg, then
  `setFabInspectorActive(isInspectorActive())`.
- **`shouldIgnore(el)` in dom-inspector.js** must include `hh-ar-fab-inspector`
  ID or the inspector will highlight its own toggle button when hovering.
  This was already added in v1.9.61.0 — do not remove.
- **AHG submodule pointer changes silently** if you run `git add -A` after
  any operation that touched `anti-hallucination-guard/`. Always check
  `git status` before committing and `git restore --staged anti-hallucination-guard`
  if it appears staged without intent.
- **`cascade/state.json`** (in `cascade/` subdir) is a DIFFERENT file from
  root `cascade-state.json`. Both exist. The root one is the source of truth.
  The `cascade/` one is a generated artifact. Do not confuse them.
- **Emoji is FORBIDDEN** in source files (Rule 15, see
  `docs/UNICODE_POLICY-v2.1.md`). Use inline SVG for icons. CHANGELOG
  entries also must not contain emoji.
- **Em-dash "--" is discouraged** (Rule 15). Use `--` (double hyphen) instead.
  The pre-commit hook will warn.
